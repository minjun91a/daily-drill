# 🐧 Linux L4 — 권한·소유권 (1·2일차) (2026-07-09~10 · WSL Ubuntu 24.04)

> **1일차**: rwx 읽기 · `chmod`(기호) · 파일 vs 폴더의 `x`. **2일차(↓ 아래)**: 숫자(8진수) 표기(644·755)·"절대 설정" 함정·`chown`.

**개념 씨앗**: `ls -l` 맨 앞 10글자가 권한. **1글자**(종류: `-`파일·`d`폴더·`l`링크) + **3+3+3**(주체별 권한). 주체 3 = **u**(주인=user)·**g**(그룹)·**o**(나머지=others). 권한 3 = **r**(읽기)·**w**(쓰기)·**x**(실행), 없으면 `-`. `chmod [u/g/o/a][+/−][rwx] 파일`로 권한을 켜고/끈다.

## 예측 A — 권한 문자열 읽기  → **파일 읽기 정확**
`deploy.sh` = `-rwxr-xr-x` 를 풀기: `-`(파일) / 주인 `rwx`(다 됨) / 그룹 `r-x`(읽기+실행) / 나머지 `r-x`(읽기+실행). **정확.** `report.txt`(`-rw-r--r--`)와의 차이 = **`x`(실행) 자리 전부** → deploy.sh는 실행 가능한 스크립트, report.txt는 데이터. **`x` = "실행 가능한 프로그램" 표시**(스크립트가 permission denied면 대개 x가 꺼진 것).

## 예측 B — `chmod o-r report.txt`  → **❌ o를 주인으로 착각**
- **문제**: `report.txt` = `-rw-r--r--` 에서 `chmod o-r` 실행 후 권한은?
- **내 예측 (그대로)**: "주인의 읽기를 뺀다 즉, `--w-r--r--` 이렇게 된다."
- **실제** (WSL 실행): `-rw-r-----`. **나머지(others, 마지막 3칸)**의 `r`이 빠짐 — 주인이 아님.
  ```
  예측:  - -w-  r--  r--     (주인 첫 칸의 r을 뺐다고 봄)
  실제:  - rw-  r--  ---     (나머지 마지막 칸의 r이 빠짐)
  ```
- **교정 (개념)**: `o`는 owner가 아니라 **others(나머지)**. 주인은 **`u`**(user). 짝 = `u`주인 / `g`그룹 / `o`나머지 / `a`전체. 글자가 owner처럼 생겨 다들 걸리는 함정.
- **교훈**: **`u`=user=주인, `o`=others=나머지.** `chmod`에서 사람을 겨냥할 땐 이 매핑을 먼저 확정한다(o를 주인으로 쏘면 정반대 칸이 바뀐다).

## 재확인 — `chmod u-w report.txt`  → **✅ 사람 정확히 겨냥**
`-rw-r-----` 에서 `chmod u-w` → 예측 `-r--r-----` (주인 u의 쓰기 w만 끔). 실제도 `-r--r-----`. **일치.** u/g/o 겨냥 교정됨.

## 검증 함정 — 폴더에서 `x` 빼기  → **❌ 개념을 정반대로 알고 있었음**
- **문제**: `logs`(`drwxr-xr-x`, 안에 `app.log`)에서 `chmod a-x logs`(→ `drw-r--r--`) 후, ① `cat logs/app.log` ② `cd logs` 가 되나?
- **내 예측 (그대로)**: "1. 폴더 안 파일 내용을 볼 수 있다. 2. 폴더 안으로는 들어갈 수 있다." (둘 다 "된다")
- **실제** (WSL 실행): **둘 다 Permission denied** — 심지어 주인(min)인데도.
  ```
  drwxr-xr-x (정상):  ls ✅ / cat ✅ 로그 한 줄 / cd ✅ 들어감
  drw-r--r-- (x 제거):  ls ✅ app.log(이름만) / cat ❌ Permission denied / cd ❌ Permission denied
  ```
- **자기 교정 (원문)**: "아 x가 없어야 문이 열리는 줄 알았어. 완전히 반대로 개념 이해했었네." → **폴더 `x` = 들어갈 수 있는 "문". 빼면 잠긴다.**
- **교훈**: 폴더에서 **`r`과 `x`는 다른 일**. `r`=안의 **이름 목록** 보기(`ls`) / `x`=**들어가서 내부 파일에 접근**(`cd`·`cat`·열기). `x` 없으면 "이름은 보이나 손 못 대는" 상태. **`x` 의미: 파일=프로그램 실행 / 폴더=들어가기(통과).**

## 직접 — 잠긴 폴더 다시 열기
**스펙**: `chmod a-x`로 잠긴 `logs`(`drw-r--r--`)에 주인이 다시 `cd`할 수 있게 고쳐라(남에겐 열지 말 것).

**① ✅ 합격**
```bash
chmod u+x logs      # 주인(u)에게만 x를 다시 켬 → drwxr--r--
```
**실행 결과** (WSL): `drw-r--r--`(cd 거부) → `chmod u+x` → `drwxr--r--` → `cd logs` ✅, `cat logs/app.log` ✅ "로그 한 줄".
**판단 (원문)**: "굳이 a로 모두에게 권한을 줄 이유가 없다." → `a+x`도 풀리지만 그룹·나머지에 불필요하게 문을 엶. 주인만 필요하니 `u+x`. = **최소 권한(least privilege)** 원칙.

## 그날의 핵심
- **`u`=user=주인, `g`=group, `o`=others=나머지, `a`=all.** `o`는 owner가 아니다 — `chmod`에서 사람을 잘못 겨냥하면 정반대 칸이 바뀐다.
- **폴더의 `x` = 들어가기(cd·통과·내부 접근), 실행이 아니다.** `x`를 빼면 주인이라도 `cd`·`cat`이 막힌다. `r`(이름 목록)과는 별개.
- **파일 `x` = 실행 가능 표시**(스크립트가 안 돌면 x부터 의심).
- **권한은 최소로**: 주인만 필요하면 `u+`, 굳이 `a+`로 남에게 열지 않는다(least privilege).
- (2일차 ↓에서 다룸) 숫자 표기 `644`/`755`(r=4·w=2·x=1) + `chown`(소유자 변경).

# 🐧 L4 2일차 — 숫자(8진수) 표기 + chown (2026-07-10)

**개념 씨앗**: rwx를 숫자로 — **r=4 · w=2 · x=1**, 한 주체(주인/그룹/기타)의 권한 = 켠 것의 합(0~7). 세 자리 = **주인·그룹·기타** 순(예 `644`=rw-r--r--). **기호(`chmod u+x`)는 상대 변경**(하나 켜고/끄기)이지만 **숫자(`chmod 644`)는 절대 설정**(세 자리 통째 교체). 표준: 파일 `644`·폴더/실행 `755`·비밀 `600`. `chown 사용자:그룹 파일` = **소유자** 변경(보통 `sudo`).

## 예측 — 숫자 ↔ 문자  → **A·B·함정 전부 힌트 없이 통과**
| # | 문제 | 내 예측 | 실제(WSL) | |
|:-:|---|---|---|:-:|
| A | `chmod 640` 후 문자열 | `-rw-r-----` | `-rw-r-----` | ✅ |
| B | `-rwxr-x---` → 숫자 | `750` | `750` | ✅ |
| 함정 | `755` 스크립트에 `chmod 600` 후, 문자열 & `./` 실행? | `-rw-------` · 실행 불가(x 잠금) | `-rw-------` · `Permission denied` | ✅ |

- **함정 핵심**: **숫자는 절대 설정** → `755`의 `x`·그룹·기타 권한이 `600`으로 **싹 덮여** 사라짐. 주인이라도 `x`가 없어 실행 불가. (기호 `+/−`는 상대라 이런 전면 교체가 없다 — "600은 주인 rw 추가"가 아니라 "전체를 rw-------로 덮음".)

## 직접 — 숫자 chmod & chown

**① 숫자 chmod** — 스펙: `tool.sh` 주인 `r-x` · 그룹 `r--`(읽기만) · 기타 `---`
- ❌ 1차 `chmod 520` → `-r-x-w----` : 그룹에 **읽기(4) 대신 쓰기(2)** 를 줌(정반대). **읽기=4, 쓰기=2** 혼동.
- ✅ 2차 `chmod 540 tool.sh` → `-r-xr-----` : 그룹 `4=r--` 정확. (숫자만 쓰면 안 됨 — **파일명 필수**.)

**② chown** — 스펙: `data.db` 소유자 `alice` · 그룹 `dev`
- ❌ 1차 `chown alice:dev sudo data.db` : `sudo`를 인자로 오해(파일 2개처럼 취급).
- ✅ 2차 `sudo chown alice:dev data.db` : **`sudo`는 명령 맨 앞 접두사**(뒤 명령을 관리자로 실행). 형식 `chown 소유자:그룹 파일`.

## 그날의 핵심 (2일차)
- **숫자 권한 = r4·w2·x1의 자리별 합.** "읽기만"은 **4**(쓰기 2와 헷갈리지 말 것 — 오늘 여기서 걸림).
- **숫자는 절대 설정(통째 교체) · 기호는 상대 변경.** `chmod 600`은 기존 `x`·그룹·기타 권한을 **다 날린다**. 실행되던 스크립트가 안 돌면 숫자 chmod로 x를 덮었는지 의심.
- **`chown 사용자:그룹 파일`** 로 소유자 변경, 보통 **`sudo`를 맨 앞**에. (`sudo`는 인자가 아니라 접두사.)
