# 📅 2026-07-08 · Day 7 — SQL

## 오늘의 문제 (LEFT JOIN + GROUP BY · 빈 그룹 집계)

- **문제 (검토한 코드)**:
  ```sql
  SELECT g.title,
         COUNT(p.id)   AS purchase_count,
         SUM(p.amount) AS total
  FROM games g
  LEFT JOIN purchases p ON p.game_id = g.id
  GROUP BY g.title;
  ```
  데이터: **games** = 네온레이스(1)·스타폴(2)·문라이커(3). **purchases** = 네온레이스 2건(5000·3000)·스타폴 1건(9900)·**문라이커 0건**.
- **내 예측 (그대로)**: "1. 4개 행. 2. title = 네온레이스, 네온레이스, 스타폴, 문라이커 / purchase_count = 2, 1, NaN / total = 17900." → **count 2·1은 정확**, 그러나 **행 수·문라이커·total 3곳 어긋남.**
- **실제** (sqlite3 실제 실행): **3행.** 네온레이스 `count 2 / total 8000` · 스타폴 `count 1 / total 9900` · 문라이커 `count 0 / total NULL`.

### 어긋난 3곳 (전부 GROUP BY 관련)

**① 4행이 아니라 3행 — `GROUP BY`가 다시 접는다**
JOIN 직후 중간 결과는 진짜 4행(네온레이스+5000, 네온레이스+3000, 스타폴+9900, 문라이커+NULL)이 맞다 — Day 4의 fan-out을 정확히 트레이스함. 그러나 마지막 줄 `GROUP BY g.title`이 같은 title을 1행으로 접어 **출력은 title 종류 수 = 3행.** "늘리는(JOIN)" 단계는 봤지만 "접는(GROUP BY)" 단계를 안 적용.

**② 문라이커 count는 NaN이 아니라 `0` — total이 NULL**
문라이커는 0건 → LEFT JOIN이 `(문라이커, NULL)` 행 1개 생성. 이때:
- `COUNT(p.id)` → NULL은 세지 않음 → 셀 게 없어 **0** (COUNT는 절대 NULL을 안 뱉음)
- `SUM(p.amount)` → 더할 값이 없으면 **NULL** (0 아님)

→ NaN(=NULL)을 count에 붙였는데, NaN은 **total 쪽**이 정답. **자리를 바꿔 냈다.** 외울 것: **빈 그룹에서 COUNT는 0, SUM은 NULL.**

**③ total 17900(하나) → 그룹별로 쪼개짐**
17900 = 전체 합. 그러나 `GROUP BY title`이 있으면 SUM도 각 title 안에서만 계산 → 8000 / 9900 / NULL. GROUP BY는 "행을 그룹으로 나눠 각 칸 안에서 집계"라 집계값은 그룹마다 하나.

- **교훈**: `GROUP BY` = **접기 + 그룹별 집계**. JOIN으로 늘어난 행을 그룹당 1행으로 접고, 집계는 그룹 안에서만. 빈 그룹은 **COUNT→0, SUM→NULL**(0으로 보이려면 `COALESCE(SUM(...), 0)`).

## 직접 구현 (합격 후 · 검증 → 생산)

### 직접 구현 — LEFT JOIN + 빈 그룹 0 처리 쿼리 백지에서 짜기
**스펙**: `streamers`·`donations`에서 **① 이름 ② 후원 건수 ③ 후원 총액**을 출력. **후원 0건 스트리머도 결과에 포함**하고, 그 **총액은 NULL이 아니라 0**으로. 백지 작성.

| 스트리머 | 후원 | 정답 `count / total` |
|------|------|:---:|
| 아리 | 5000·10000 | `2 / 15000` |
| 벤투 | 3000 | `1 / 3000` |
| 카이 | 0건 | `0 / 0` |

**내 구현 변천 (원문 그대로)** — `FROM/LEFT JOIN/GROUP BY` 뼈대는 4번 모두 동일, **SELECT 집계만** 바뀜:

**① 1차 — ❌ 카이가 `1 / NULL` (스펙 위반 2곳)**
```sql
SELECT s.name, count(*) as donation_count, sum(d.amount) as total
FROM streamers s
LEFT JOIN donations d ON d.streamer_id = s.id
GROUP BY s.name;
```
`COUNT(*)`는 **행**을 셈 → 카이는 LEFT JOIN이 만든 `(카이, NULL)` 행이 1개 존재하므로 **1**(0 아님). `SUM`은 COALESCE 없어 카이 **NULL**.

**② 2차 — ⚠️ count는 고쳤으나 아리 total이 `5000` (SUM 증발)**
```sql
SELECT s.name, count(d.id) as donation_count, coalesce(d.amount, 0) as total
FROM streamers s
LEFT JOIN donations d ON d.streamer_id = s.id
GROUP BY s.name;
```
`COUNT(d.id)` → 카이 0 ✅. 그러나 COALESCE로 감싸며 **`SUM`을 지움** → `d.amount` 혼자는 그룹의 여러 행을 못 합쳐 SQLite가 아리 2행 중 아무거나 하나(5000)를 집어옴. count 2인데 total 5000 = **모순 신호.**

**③ 3차 — ❌ 실행 에러 (괄호 위치)**
```sql
SELECT s.name, count(d.id) as donation_count, coalesce(sum(d.amount, 0)) as total
FROM streamers s
LEFT JOIN donations d ON d.streamer_id = s.id
GROUP BY s.name;
```
`wrong number of arguments to function sum()`. SUM은 인자 1개만 받는데 `, 0`이 sum 괄호 안으로 들어감. `0`은 COALESCE 것이지 SUM 것이 아님.

**④ 4차 — ✅ 합격**
```sql
SELECT s.name, count(d.id) as donation_count, coalesce(sum(d.amount), 0) as total
FROM streamers s
LEFT JOIN donations d ON d.streamer_id = s.id
GROUP BY s.name;
```
**실행 결과** (sqlite3 실제 실행)
```
name | donation_count | total
아리  | 2              | 15000
벤투  | 1              | 3000
카이  | 0              | 0
```
→ 스펙 일치. `SUM`을 먼저 닫고(`sum(d.amount)`) 그 바깥을 `COALESCE(..., 0)`로 감쌈.

**교훈**
- `COUNT(*)`는 **행**을 세고 `COUNT(컬럼)`은 **NULL 아닌 값**을 센다. LEFT JOIN에서 빈 쪽을 0으로 세려면 반드시 `COUNT(조인테이블.컬럼)`.
- `COALESCE`는 "NULL이면 대신 쓸 값"일 뿐 **합치지 못한다.** 합치는 건 `SUM`의 일 → 둘 다 필요하고 순서는 **SUM 먼저, COALESCE로 겉을 감싼다**: `COALESCE(SUM(x), 0)`.
- 함수 중첩은 **괄호 짝**이 곧 인자 소속. `sum(x, 0)` vs `sum(x), 0` — `, 0`이 어느 괄호 안에 있느냐로 SUM/COALESCE 것이 갈린다.
- `count 2인데 total 5000` 같은 **내부 모순**은 버그를 잡는 신호다 — 건수와 합계가 앞뒤 안 맞으면 집계가 틀린 것.

## 그날의 핵심
- **`GROUP BY` = 접기 + 그룹별 집계.** JOIN으로 늘어난 행을 그룹당 1행으로 접고, SUM/COUNT는 각 그룹 안에서만 계산 → 집계값은 그룹마다 하나(전체 합 하나 아님).
- **빈 그룹(LEFT JOIN 미매칭): COUNT→0, SUM→NULL.** NaN이 붙는 쪽은 COUNT가 아니라 SUM. 0으로 보이려면 `COALESCE(SUM(...), 0)`.
- **`COUNT(*)` ≠ `COUNT(컬럼)`.** `*`는 행을 세서 빈 쪽도 1로 셈, 컬럼은 NULL 아닌 값만 세서 0. LEFT JOIN 집계엔 컬럼을 센다.
- **(직접 구현)** COALESCE는 합치지 못한다 — SUM으로 합친 뒤 COALESCE로 감싼다. 괄호 짝이 인자 소속을 정한다. 건수·합계의 내부 모순은 버그 신호.
