# 📅 2026-07-14 · Day 11 — pandas

## 오늘의 문제 (chained indexing = 복사본 · 대입이 원본에 안 닿는다)

- **문제 (검토한 코드)**: 레벨 10 미만 유저(A·C)의 `coins`를 0으로 만들려는 코드.
  ```python
  df = pd.DataFrame({'user':['A','B','C','D'],'level':[5,12,3,20],'coins':[100,200,300,400]})
  df[df['level'] < 10]['coins'] = 0
  ```
- **내 예측 (그대로)**:
  > 1. 모두 0개로 값이 나온다.
  > 2. 동료 의도와 다르게 모두 0 coin이 되었다.
  > 3. df[(df['level'] < 10]['coin'] = 0)] 으로 수정해야 한다.
  → **정반대로 틀림.** "필터가 안 먹혀 전부 0"으로 봤으나, 실제는 **아무 행도 안 바뀜.**
- **실제** (pandas 3.0.2 실행): `coins` = `[100, 200, 300, 400]` **그대로** + **`ChainedAssignmentError`**("chained assignment never works").

### 무슨 일이 일어났나 — chained `[]`는 복사본을 만든다
```python
df[df['level'] < 10]      # ① 원본이 아니라 새로 걸러낸 "복사본(임시 DataFrame)"
              ['coins'] = 0 # ② 그 복사본의 coins에 0 대입 → 복사본은 다음 줄로 넘어가며 버려짐
```
원본 `df`는 그 대입을 **본 적도 없다** → 전부 그대로. `[]`를 **두 번 연달아** 쓰면(chained indexing) 거의 항상 이 함정. pandas 3.0은 아예 에러로 경고한다. **민준 깨달음(원문)**: *"df에 또 다른 df의 dataframe을 생성한 문법이였네."*

- **교정 (고친 코드) — 변천 원문 그대로**:

  **① 1차 — ❌ 문법 깨짐 ("괄호를 지운다" 오개념)**
  ```python
  df['level'] < 10]['coins'] = 0
  ```
  괄호 짝이 깨짐. 그리고 `df['level'] < 10`은 그냥 불리언 Series(True/False 목록)일 뿐 — 괄호를 빼는 게 아니라 **두 개의 `[]`를 하나로 합쳐야** 한다.

  **② 2차 — ❌ `TypeError: '<' not supported between 'str' and 'int'`**
  ```python
  df.loc['level' < 10, 'coins'] = 0
  ```
  `.loc[행, 열]` 구조는 맞았으나 `'level'`은 **글자 그대로의 문자열** → 숫자와 비교 불가. 열의 **실제 값**을 꺼내려면 `df['level']`.

  **③ 3차 — ❌ 에러 없이 조용히 틀림 (쓰레기 컬럼 4개 생성)**
  ```python
  df.loc[df['level'] < 10, df['coins']] = 0
  ```
  결과:
  ```
    user  level  coins  100  200  300  400   ← 100·200·300·400 컬럼이 새로 생김!
  0    A      5    100  0.0  0.0  0.0  0.0
  1    B     12    200  NaN  NaN  NaN  NaN
  ```
  `df['coins']`는 값 목록 `[100,200,300,400]`. `.loc`의 **열 자리**에 넣으니 pandas가 이 숫자들을 **"만들 컬럼 이름(라벨)"**으로 해석 → 쓰레기 컬럼 4개 생성, 원본 `coins`는 손도 안 댐. **에러 하나 없이.**

  **④ 4차 — ✅ 합격**
  ```python
  df.loc[df['level'] < 10, 'coins'] = 0
  ```
  **실행 결과**: `coins` = `[0, 200, 0, 400]`.

- **교훈**:
  - **chained indexing(`df[...][...] = 값`)은 복사본에 쓰고 버린다 — 원본은 안 바뀐다.** 대입문을 볼 때 "이게 원본을 바꾸나, 복사본을 바꾸나?"를 먼저 묻는다. `df[mask]`는 새 객체를 만든다.
  - 부분 행을 수정하려면 **`.loc[행 마스크, '열 라벨'] = 값`** — `df` 자체를 한 번에 인덱싱해 원본에 직접 쓴다.
  - **`.loc[행, 열]`의 두 자리는 성격이 다르다**: 행 = "어떤 행?"→**값**으로 만든 조건(`df['level'] < 10`), 열 = "어느 열?"→**이름(라벨)** `'coins'`(문자열, `df['coins']` 아님). 한 자리에서 통한 문법을 다른 자리에 그대로 이식하지 말 것.
  - **에러가 안 나도 스펙과 대조한다** (Day 8 재확인) — 3차는 에러 없이 쓰레기 컬럼을 만들었다.

## 재도전 (같은 개념 · 다른 유형) — ✅ 힌트 없이 합격

- **문제**: 60점 미만(X·Z)을 60으로 끌어올리려는 코드. `scores[scores['score'] < 60]['score'] = 60`
- **내 예측 (그대로)**:
  > 1. [45, 80, 30] 그대로다.
  > 2. 의도대로 안됐다.
  > 3. 우선 scores 라는 dataframe 을 loc 로 배열을 나열하지 않고 대괄호[] 를 한번 더 씀으로써 새로운 dataframe을 만들어 새로운곳에 적용했기 때문에.
  → **예측 3개 전부 정확** (특히 ③ 메커니즘 = 복사본). 오늘의 문제에서 정반대로 틀린 걸 완전히 뒤집음.
- **실제**: `score` = `[45, 80, 30]` 그대로 + `ChainedAssignmentError`.
- **교정 (스스로 먼저 제시)**: `scores.loc[scores['score'] < 60, 'score'] = 60` → `[60, 80, 60]`. **힌트 없이 정확.**

## 직접 구현 (합격 후 · 검증 → 생산)

### 직접 구현 — 부분 행 in-place 수정 백지에서 짜기
**스펙**: 게임 상점 재고표 `products`에서 **재고(`stock`)가 0인 상품의 `price`를 `-1`(품절 표시)로**. 나머지 행은 그대로. 백지 작성.

| product | stock | price | 기대 price |
|---------|:-----:|:-----:|:---:|
| Sword  | 0 | 100 | **-1** |
| Shield | 5 | 80  | 80 |
| Potion | 0 | 20  | **-1** |
| Bow    | 3 | 150 | 150 |

**내 구현 변천 (원문 그대로)**:

**① 1차 — ❌ `SyntaxError: cannot assign to subscript here. Maybe you meant '==' instead of '='?`**
```python
products.loc[products['stock'] = 0, 'price'] = -1
```
`=`(대입, "넣어라")를 마스크 자리에 씀 → `products['stock'] = 0`은 "stock 열을 전부 0으로 만들어라"라는 명령이 돼버림. 마스크는 True/False가 나오는 **비교식**이어야 함.

**② 2차 — ✅ 합격**
```python
products.loc[products['stock'] == 0, 'price'] = -1
```
**실행 결과**: `price` = `[-1, 80, -1, 150]` → 스펙 일치.

**교훈**:
- **`=`(대입) vs `==`(비교)**: 행 마스크는 "이게 저것과 같니?"를 각 행마다 물어야 하므로 `==`. `=`는 값을 넣는 명령이라 마스크 자리에선 SyntaxError. (모든 마스크는 True/False가 나오는 조건식 — `< `, `==` 등.)
- `.loc[행 마스크, '열 라벨'] = 값` 꼴을 백지에서 재현 — 인식(chained 알아보기) → 생산(직접 짜기) 전이 확인.

## 그날의 핵심
- **chained indexing(`df[mask][col] = 값`)은 복사본에 쓰고 버린다 — 원본은 안 바뀐다** (pandas 3.0은 `ChainedAssignmentError`). 부분 행 수정은 **`df.loc[행 마스크, '열 라벨'] = 값`** 한 방으로.
- **`.loc[행, 열]`의 두 자리는 성격이 다르다**: 행=값으로 만든 조건(`df['x'] < n`), 열=이름 라벨(`'col'`). 한 자리 문법을 다른 자리에 이식하다 3차에서 쓰레기 컬럼을 만들었다(에러 없이).
- **(직접 구현)** 마스크는 `==`(비교)이지 `=`(대입)이 아니다. 에러가 안 나도 결과를 스펙과 대조한다.
