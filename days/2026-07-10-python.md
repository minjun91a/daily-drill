# 📅 2026-07-10 · Day 9 — Python

## 오늘의 문제 (중첩 리스트 `*` 복제 = 참조 공유)

- **문제 (검토한 코드)**:
  ```python
  # 3x3 게임 보드를 0으로 초기화
  board = [[0] * 3] * 3
  board[1][1] = 9        # 가운데(1,1)에만 9를 놓으려는 의도
  print(board)
  ```
- **내 예측 (그대로)**: "1. `[[0,9,0],[0,9,0],[0,9,0]]`. 2. 각 배열에 모두 가운데 9가 들어간다." → ✅ **힌트 없이 통과**. 순진한 답(`[[0,0,0],[0,9,0],[0,0,0]]`)이 아니라 **실제 동작(세 행 다 9)** 을 정확히 예측.
- **실제** (python 실행): `[[0,9,0],[0,9,0],[0,9,0]]`. `id` 확인 → 세 행 id **동일**, `board[0] is board[1]` → **True**. **세 행이 "같은 리스트 하나를 3번 참조"** 하는 것.
- **왜**: `[0]*3`이 리스트 하나 `[0,0,0]` 생성 → `[...] * 3`은 그걸 **복사하지 않고 참조를 3번 나열**. 그래서 그 하나를 `board[1][1]=9`로 바꾸면 셋 다 바뀐 것처럼 보임(실은 하나).
- **🔗 Day 3와 한 뿌리**: 함수 기본값 `[]` 공유와 같은 계열 — **파이썬의 `*`·기본값·`b=a` 대입은 객체를 복사하지 않고 참조만 늘린다.**

## 직접 구현 (의도대로 — 가운데만 9)
**스펙**: `board`를 **세 행이 독립된 리스트**가 되게 만들어 `board[1][1]=9`가 가운데만 바꾸도록. 목표 `[[0,0,0],[0,9,0],[0,0,0]]`.

**① 1차 — ❌ 괄호는 아무것도 안 바꿈**
```python
board = ([[0] * 3]) * 3
```
`([[0]*3])`는 그냥 `[[0]*3]`를 묶은 것(콤마 없어 튜플도 아님) = **같은 객체**. 문제의 `* 3`이 그대로라 여전히 세 행 공유(`board[0] is board[1]` → True). **곱하기는 참조를 복제할 뿐, 어디 괄호를 씌워도 독립 리스트를 못 만든다.**

**② 2차 — ✅ 리스트 컴프리헨션**
```python
board = [[0]*3 for i in range(3)]
```
**실행 결과** (python 실제 실행)
```
[0, 0, 0]
[0, 9, 0]
[0, 0, 0]
board[0] is board[1] → False   (id 셋 다 다름 = 독립)
```
반복마다 `[0]*3`을 **새로 실행** → 매번 다른 리스트 생성 → 세 행 독립. (Day 6에서 쓴 컴프리헨션이 여기서도 정답.)

## 그날의 핵심
- **`[[x]*n] * m`은 안쪽 리스트를 공유한다** — `* m`이 참조를 복제하기 때문. 2D 리스트를 이 방식으로 만들면 **한 칸 수정이 전 행에 번진다**.
- **독립된 행은 컴프리헨션으로**: `[[x]*n for _ in range(m)]` — 반복마다 새 리스트 생성.
- **파이썬 복사 감각**: `*`·함수 기본값·대입(`b=a`)은 **복사가 아니라 참조 증가**. "따로인 줄 알았는데 하나" 계열(Day 3·Day 9 공통). 진짜 복사가 필요하면 컴프리헨션·`copy`·`[:]` 등.
- **예측 성과**: 순진한 답 대신 실제 공유 동작을 힌트 없이 예측 = "코드를 위→아래로 읽지 않고 **객체 모델로 굴리는**" 검증이 되기 시작.
