# 📅 2026-07-20 · Day 12 — Python

## 오늘의 문제 (falsy 함정 · `if not x:`는 `0`·`[]`도 같이 잡는다)

- **문제 (검토한 코드)**: 할인율을 안 주면 기본 10%를 적용하려는 함수.
  ```python
  def apply_discount(price, discount=None):
      if not discount:
          discount = 10          # 할인율 미지정 시 기본 10%
      return price * (1 - discount / 100)

  print(apply_discount(1000))       # ① 할인율 안 줌     → 기대 900
  print(apply_discount(1000, 20))   # ② 20% 할인         → 기대 800
  print(apply_discount(1000, 0))    # ③ 0% 명시(정가)    → 기대 1000
  ```
- **내 예측 (그대로)**:
  > 셋다 900 으로 출력
  > 동료 의도대로 안나오고,
  > 함수에 discount=None 을 디폴트값으로 설정해놨기에
- **실제** (실행): **900.0 / 800.0 / 900.0** — ②가 800(예측 900), ③이 900(기대 1000 = 버그).

### ②가 800인 게 결정적 증거 — `None` 기본값은 무죄
`discount=None` 기본값을 원인으로 지목했으나, ②에서 20을 넘기니 20을 그대로 써서 800이 나옴 → **기본값 설정은 정상 작동**. 진짜 범인은 `if not discount:` 한 줄.

```
discount=None  → not discount = True   → 기본 10%로 덮어씀   (의도대로)
discount=20    → not discount = False  → 준 값 그대로 사용    (의도대로)
discount=0     → not discount = True   → 기본 10%로 덮어씀   ← 버그
```

`not 0`도 **True**라서, 이 조건문은 **"안 줬다(None)"와 "0을 줬다"를 구분하지 못한다.** 손님이 할인 0%를 **명시**했는데 함수가 10%를 얹어 정가 1000원짜리를 900원에 팔아버린다 — **에러 없이 조용히.**

**Python falsy 값**: `None`, `0`, `0.0`, `""`, `[]`, `{}`, `False` → `if not x:`는 이걸 **전부 한 그물로** 잡는다. "값이 없나?"를 물으려던 게 실제론 "없거나·0이거나·비었나?"를 묻고 있다.

- **교정 (고친 코드)**:

  **① 1차 — ❌ (막힘)**: `if not x:` 라고만 적음 — 고쳐야 할 패턴 자체를 그대로 씀. Day 3의 `is None` 규칙을 힌트로 제시.

  **② 2차 — ✅ 합격**
  ```python
  def apply_discount(price, discount=None):
      if discount is None:
          discount=10
      return price * (1 - discount / 100)
  ```
  **실행 결과**: `900.0 / 800.0 / 1000.0` → ③이 정가로 복구.

- **교훈**:
  - **`if not x:`는 "값이 없나?"가 아니라 "falsy인가?"를 묻는다.** `0`·`""`·`[]`가 **정상 값일 수 있는 자리**에서 쓰면 그 값을 "없음"으로 오인해 조용히 덮어쓴다.
  - "정말 안 준 경우"만 잡으려면 **`if x is None:`** — Day 3의 "`None` 비교는 `is None`"이 *비교*뿐 아니라 **존재 확인**에도 그대로 적용된다.
  - **반례 훈련**: `if not x:`를 볼 때마다 "x가 0이면? 빈 문자열이면? 빈 리스트면?"을 굴려본다. 그게 정상 값일 수 있으면 그 조건문은 버그다.

## 재도전 (같은 개념 · 다른 유형) — ✅ 힌트 없이 합격

- **문제**: 수신자를 안 주면 관리자에게 보내려는 함수. `if not recipients: recipients = ["admin@company.com"]`. 호출 = ① 미지정 ② 2명 지정 ③ `[]`(= "아무에게도 보내지 마라").
- **내 예측 (그대로)**:
  > 1. 관리자, 2명, 관리자
  > 2. 의도와 다르다.
  > 3. 3 빈 목록을 인지 못하여 관리자에게 보내게 된다.
  > if recipients is None: 으로 수정해야 1번은 그대로 인지되어 관리자로 보내지고, 3번은 빈 목록만 출력하게 된다.
  → **출력 3개·원인·교정 전부 정확.** 교정까지 스스로 먼저 제시.
- **실제**: 원본 `1명 / 2명 / 1명` (③이 버그) → 교정 `1명 / 2명 / 0명`.
- **핵심**: 숫자 `0`에서 배운 falsy 함정을 **빈 리스트 `[]`로 전이**시킴 — 둘 다 falsy라는 같은 뿌리를 봄. 오늘의 문제에서 ②를 틀리고 원인을 오진단했던 걸 완전히 뒤집음.

## 직접 구현 (합격 후 · 검증 → 생산)

### 직접 구현 — falsy 함정 피해 기본값 함수 백지에서 짜기
**스펙**: `calc_total(price, shipping_fee=None)` — 배송비 미지정 시 기본 **3000원**, 주면 그 값, **`0`을 명시하면 무료배송**(price 그대로). 백지 작성, 함정 경고 없음.

| 호출 | 기대 |
|---|:---:|
| `calc_total(10000)` | 13000 |
| `calc_total(10000, 2500)` | 12500 |
| `calc_total(10000, 0)` | 10000 |

**내 구현 (원문 그대로) — ✅ 1회 만에 합격**
```python
def calc_total(price, shipping_fee=None):
    if shipping_fee is None:
        shipping_fee = 3000
    return price + shipping_fee
```
**실행 결과**: `13000 / 12500 / 10000` → 스펙 일치.

**교훈**: 함정 경고 없이 백지에서 **`is None`을 스스로 선택** — "기본값이 필요한 자리엔 `not`이 아니라 `is None`"이 판단 기준으로 자리잡음. 인식(버그 알아보기) → 생산(처음부터 안 만들기) 전이 완료.

## 그날의 핵심
- **`if not x:`는 `None`·`0`·`""`·`[]`·`{}`·`False`를 전부 한 그물로 잡는다.** "값이 없나?"를 물으려던 자리에 쓰면 **정상 값 `0`/빈 컨테이너를 '없음'으로 오인**해 조용히 덮어쓴다(에러 없음 → 더 위험).
- **"정말 안 준 경우"만 잡으려면 `if x is None:`** — Day 3의 `is None` 규칙이 *비교*를 넘어 **존재 확인**까지 확장.
- **`0`과 `[]`는 같은 뿌리(falsy).** 숫자에서 배운 함정이 빈 리스트에도 그대로 — 재도전에서 이 전이를 힌트 없이 해냄.
- **반례 반사신경**: `if not x:` → "x가 0이면? 빈 값이면? 그게 정상 값일 수 있나?"
