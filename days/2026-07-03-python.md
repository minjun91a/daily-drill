# 📅 2026-07-03 · Day 6 — Python

## 오늘의 문제

### 오답 — 리스트를 돌면서(for) 그 리스트를 줄이면(remove) 원소를 건너뛴다
- **문제 (검토한 코드)**: 짝수를 전부 제거하려는 코드.
  ```python
  nums = [1, 2, 4, 6, 3]
  for n in nums:
      if n % 2 == 0:
          nums.remove(n)
  print(nums)
  ```
  의도: 짝수 다 제거 → `[1, 3]`.
- **내 예측 (그대로)**: "`nums = [1, 3]`. 의도대로 출력된다고 판단." → ❌ (실제 `[1, 4, 3]`)
- **실제**: `[1, 4, 3]` — **짝수 4가 살아남음.** `for`는 내부적으로 index(몇 번째)로 돈다:
  ```
  index 0: nums[0]=1 (홀수, 유지)        nums=[1,2,4,6,3]
  index 1: nums[1]=2 (짝수→remove)       nums=[1,4,6,3]   ← 2가 빠지며 4·6·3이 앞으로 당겨짐
  index 2: nums[2]=6                      ← 4는 index 1로 밀렸는데 루프는 이미 index 2 → 4를 다시 안 봄
  index 3: 범위 밖 → 종료
  ```
  4는 "홀수로 판정"된 게 아니라 **검사 자체를 건너뛴 것**(index가 이미 지나감). 돌고 있는 리스트를 그 자리에서 줄이면 인덱스가 밀려 원소를 건너뛴다.
- **교정 (고친 코드)** — 변천:

  **① 1차 — ❌ (print는 안 지움 + remove가 루프 밖)**
  ```python
  for n in nums:
      if n % 2 == 0:
          print(nums)     # 출력만, 지우지 못함
  nums.remove(n)          # 루프 밖 → 끝나고 1번, n=마지막값 3만 제거
  # 결과 [1, 2, 4, 6]  (짝수 다 남고 엉뚱하게 3 제거)
  ```
  **② 2차 — ✅ 리스트 컴프리헨션 (새 리스트)**
  ```python
  nums = [1, 2, 4, 6, 3]
  nums = [n for n in nums if n % 2 != 0]   # 홀수인 n만 골라 새 리스트
  print(nums)                              # [1, 3]
  ```
  다른 정석: 사본을 돈다 → `for n in nums[:]:` (사본 순회 + 원본 remove).
- **교훈**:
  - **돌고 있는 리스트를 그 자리에서 바꾸지(줄이지) 마라** — 원소가 앞으로 당겨지며 index가 밀려 건너뛴다.
  - 해법: (a) 사본을 돈다 `nums[:]`, 또는 (b) 조건 맞는 것만 골라 **새 리스트**(컴프리헨션). 실무는 (b)가 정석.
  - `print`는 보여줄 뿐 안 지운다(`remove`가 지움). **들여쓰기가 "루프 안/밖"을 가른다** — remove가 밖이면 딱 1번만 실행.

## 그날의 핵심
- 리스트를 순회하며 동시에 그 리스트를 수정하면 원소를 건너뛴다(for의 내부 index가 밀림).
- 안전한 길: **사본 순회(`nums[:]`)** 또는 **새 리스트 생성(컴프리헨션 `[n for n in nums if ...]`)**.
- 목표를 이루는 "일꾼"이 어느 줄인지 봐라 — `remove`가 지우고 `print`는 보여줄 뿐. 들여쓰기 = 실행 범위.
