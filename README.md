# 매일 코딩 검증 드릴 (daily-drill)

AI가 짠 코드를 **읽고 / 흐름을 따라가고 / 틀린 곳을 짚고 / 왜 그런지 설명하는** 능력을 기르기 위한 하루 한 문제 드릴. 타이핑·문법 암기가 아니라 **검증·디버깅하는 눈**을 키우는 게 목표다.

## 왜 하는가
구현을 AI에 위임해오면서 라인 단위 코드를 "읽고 검증"하는 근육이 약해졌다. 이 드릴은 그 갭을 메운다. 동시에 AI가 **"자신 있게 틀리는" 패턴**을 모아, 멀티에이전트 오케스트레이션 엔진 `mint-agent`의 trust layer 검증 규칙 후보이자 포트폴리오 자산으로 쌓는다.

## 매일 루프
1. **출제** — "AI/동료가 짠 코드인데 문제가 있다" 형태. 영역은 `SQL → pandas → Python` 순환.
2. **예측 우선** — 코드를 *돌리기 전에* 결과/문제점을 글로 먼저 적는다. (가장 중요, 생략 금지)
3. **대조** — 실제 실행 결과와 예측을 맞춰본다. 어긋난 지점이 학습 포인트.
4. **검증** — 어긋난 곳만 깊게 판다.
5. **로그** — 틀린 것만 [`LOG.md`](LOG.md)에 기록한다.
