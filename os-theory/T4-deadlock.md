# 🧠 OS 이론 T4 — 데드락 (Deadlock) (2026-07-14)

**개념 씨앗**:
- **데드락(교착상태)**: 여러 프로세스/스레드가 **서로가 쥔 자원을 기다리며** 아무도 진행 못 하고 영원히 멈춤. (T3의 lock을 **여러 개** 쓸 때 생김.)
- **4대 필요조건 (Coffman) — 4개가 *동시에 모두* 성립해야 데드락**: ① 상호배제(한 번에 한 명) ② 점유와 대기(쥔 채 기다림) ③ 비선점(강제로 못 뺏음) ④ 순환대기(대기가 원을 이룸).
- **대응**: 예방(조건을 구조적으로 못 생기게) · 회피(은행원=매 할당 전 안전상태 검사) · 탐지+회복.

## 오개념 검증 — 4개 판정  → **판정 4/4 정확 (C 속도착각 또 걸러냄)**
| # | 주장 | 내 판정 | 정답 | |
|:-:|---|:-:|:-:|:-:|
| A | 4조건 중 **하나만** 성립해도 데드락 발생 | ❌ | ❌ | ✓ |
| B | 예방하려면 4조건 중 **하나만 깨면** 된다 | ⭕ | ⭕ | ✓ |
| C | 은행원(회피)은 **더 빠르게 처리해 성능을 높이는** 기법 | ❌ | ❌ | ✓ |
| D | 자원에 **번호 매겨 오름차순 요청**하면 순환대기가 깨져 예방 | ⭕ | ⭕ | ✓ |

- **코드 증거** (python threading, 2-lock): 철수 fork1→fork2 / 영희 fork2→fork1 (엇갈림) → **DEADLOCK**(철수가 fork2 못 얻고 1.5s 타임아웃). 둘 다 fork1→fork2 (순서통일) → **정상**(아무도 안 막힘).
- **A (❌ 정확)**: 4개가 **동시에 모두** 성립해야 데드락. 하나만으론 불가 → 그래서 예방은 하나만 깨면 됨(B와 짝).
- **C (❌ 정확 — 속도착각 2연속 극복)**: "성능 향상이 아니라, 데드락으로 갈 것을 미리 파악해 막는 기법"이라 정확히 판정. **T3(mutex=빠름)에 이어 "더 낫다→빠름" 착각을 또 스스로 걸러냄.** (정밀화: 은행원은 오히려 검사비용·보수적 할당으로 **느려질 수 있다**.)
  - **면접 정밀화**: 은행원은 엄밀히 **예방(prevention)이 아니라 회피(avoidance)**. 예방=4조건을 구조적으로 차단 / 회피=조건은 허용하되 매 할당마다 "안전상태 유지되나" 계산해 위험하면 거부(미래 최대요구량 필요).
- **D (⭕ 정확, 이유 2곳 교정)**: 판정 맞음. 단 ① **PID가 아니라 *자원*에 번호**를 매김(프로세스 아님). ② **"선착순"과 무관** — 원리는 "높은 번호 쥔 채 낮은 번호 기다리기"를 규칙으로 금지 → 사이클 불가. (데모(2)가 정확히 이것: 둘 다 낮은 fork1→높은 fork2.)

## 직접 (Feynman — 엇갈림 데드락을 4조건으로 대입)
**정직한 출발**: "아직 4조건을 정확히 정의하기엔 공부가 부족, 감이 안 온다"고 스스로 밝힘 → 정의를 표로 제공하고 **대입은 직접**. (모른다고 말하는 메타인지 = 이 훈련이 권장하는 태도.)

- **내 대입 (그대로)**:
  > 상호배제: 아니다. 하나의 자원은 한번에 한명에게만 쓸수 있어 둘이서 하나를 같이 못 씀.
  > 점유와 대기: fork1를 쥔채 (fork2를) 기다리지만 순서가 올 수 없다. 영희는 fork2가 먼저라서.
  > 비선점: 못뺏는다. 영희는 fork2가 우선순위라서.
  > 순환대기: 자원 대비점이 다르기에 순환이 이루어지지 않는다.
- **채점**: ①②③ 방향 맞음, **④ 뒤집힘(핵심 오답)**.
  - **① 상호배제 ✅**: "못 나눠 씀" = 조건 **성립**. 정확.
  - **② 점유와 대기 ✅(설명 정리)**: 이 조건은 "쥔 걸 안 놓은 채 기다리나"만 봄. 철수가 fork1 쥔 채 fork2 기다림 = 성립. "순서가 올 수 없다"는 여기서 불필요.
  - **③ 비선점 ✅(이유 교정)**: "못 뺏는다" = 성립 맞음. 단 이유는 *우선순위*가 아니라 **강제 회수 수단이 없어서**(자발적 반납만).
  - **④ 순환대기 ❌→교정**: "순환 안 됨"이라 했으나 **(1) 엇갈림에선 순환 성립**(그래서 데드락). 철수→(fork2 대기)→영희→(fork1 대기)→철수 = **원**. "시작 포크가 다름"은 순환을 막는 게 아니라 **원을 만드는 원인**. (아마 (2) 고친 버전과 혼동.)
- **큰 그림**: (1)에선 **4조건 전부 성립 → 데드락**. "4개 다 O여야 데드락"을 손으로 확인.
- **파트 2 — 하나 깨기 (내 아이디어 원문)**: "2개의 포크를 만들어주면 상호배제·점유와대기·비선점이 해당 안 돼 데드락 안 걸리지 않나?"
  - **유효한 실무 접근**(자원 늘려 경쟁 제거 = 돈으로 해결). 단 **상호배제는 안 깨짐**(포크 하나는 여전히 독점). 실제로 없앤 건 **"대기"** — 포크가 충분하면 아무도 안 기다림 → 점유와 **대기**의 *대기* 소멸 → 순환대기도 자동 소멸. 트레이드오프: 자원 증설은 **비용**.
  - **데모(2) 순서통일 = ④ 순환대기를 깬 것**. (내 아이디어는 ②, 순서통일은 ④ — 둘 다 유효한 서로 다른 해법.)

## 그날의 핵심
- **데드락 = 4조건(상호배제·점유와대기·비선점·순환대기)이 *동시에 모두* 성립.** 하나라도 없으면 불가 → **예방 = 하나만 깨기.**
- **순환대기**: 엇갈린 획득 순서가 대기의 **원**을 만든다. **자원에 번호 매겨 오름차순 요청**하면 원이 안 생김(데모 실측: 엇갈림=데드락, 순서통일=정상).
- **은행원 알고리즘 = 회피(avoidance), 성능 기법 아님.** 안전상태를 미리 검사해 위험한 할당을 거부(느려질 수 있음). **🔴 "더 낫다→빠름" 속도착각을 C에서 또 스스로 걸러냄**(T3 이어 2연속).
- **[복습 후보]** 4조건 **정의를 손에 잡는 첫 단계**였고 순환대기 대입이 뒤집혔음 → 다음에 정의 재도전 1회 권장. (판정은 4/4 정확.)
