# 🧠 OS 이론 T1 — 프로세스 vs 스레드 (2026-07-08)

**개념 씨앗**:
- **프로세스** = 실행 중인 프로그램. 각자 **독립된 주소공간**(code·data·heap·stack). 프로세스끼리는 **격리** — 남의 메모리 직접 못 봄(통신은 IPC).
- **스레드** = 프로세스 **안의 실행 흐름**. 한 프로세스의 스레드들은 **code·data·heap 공유**(같은 주소공간)하되, **stack·레지스터(PC)는 각자 따로**.
- 트레이드오프: 스레드는 데이터 공유·전환이 **싸지만** 같은 메모리를 동시에 만져 **충돌(race)** 위험 → [동시성 T3]. 프로세스는 **격리돼 안전**하지만 통신·전환이 **비쌈**.
- 비유: **프로세스=격리된 집, 스레드=한 집 안 사람들** — 거실(heap·전역) 공유, 각자 방(stack) 따로.

## 오개념 검증 (4개 판정)
| # | 설명 | 내 판정 | 정답 | |
|:-:|---|:-:|:-:|:-:|
| A | 스레드는 **heap·전역변수 공유** | ⭕ | ⭕ | ✓ |
| B | 스레드는 **stack도 공유** | ❌ | ❌ | ✓ |
| C | 스레드는 **CPU 연산 자체가 더 빠름** | ⭕ | **❌** | ✗ |
| D | 다른 프로세스는 **메모리 격리** | ⭕ | ⭕ | ✓ |

- **코드 증거** (python 실제 실행):
  - 스레드 2개가 같은 전역 리스트에 append → `['A','B']` **둘 다 보임** = heap 공유 (A ⭕).
  - 자식 프로세스가 `counter=999`로 바꿔도 부모는 여전히 `0` = 메모리 격리 (D ⭕).
- **C (정답 ❌)**: 내 이유 "이동루트가 더 짧다"는 **컨텍스트 스위칭이 가볍다**는 맞는 관찰. 그러나 그건 **"전환이 싸다"**지 **"연산이 빠르다"**가 아니다. 스레드든 프로세스든 **같은 CPU** → 명령어 실행 속도 동일. 스레드가 싼 것 = 생성·**전환**·통신 비용. **원인(전환 가벼움)은 맞고 결론 라벨(연산 빠름)을 잘못 붙임.**
- **🔴 핵심 패턴 (T0 재발)**: T0의 "커널 모드 = 더 빠름"(권한↔속도 혼동)과 **판박이**. 이번엔 **비용(cost) 차이를 속도(speed) 차이로** 착각. **"싸다 ≠ 빠르다."** 뭔가 "더 낫다"고 느끼면 → 정확히 뭐가 싼/빠른/안전한 건지 되묻기.

## 직접 (Feynman — 크롬은 왜 탭마다 프로세스?)
**시나리오**: 탭 A=유튜브(크래시/악성), 탭 B=은행 로그인. 스레드 vs 프로세스로 나눴을 때 차이?

- **내 답 (그대로)**: "1. 격리를 통한 개인정보 보안에 강해짐 (+ 스레드로 하면 메모리가 커져 안정성 떨어짐). 2. 프로세스는 무겁다 = 공간 용량이 커 처리·비용에 취약."
- **채점**:
  - **보안 ✅** — 격리 덕에 악성 탭이 은행 탭 메모리(비밀번호)를 못 훔친다. 정확.
  - **안정성 — 결론 ⭕, 이유 ❌**. "메모리가 커져서"가 아니다. ① 메모리 방향이 거꾸로: 스레드는 주소공간 **공유**라 오히려 메모리를 **덜** 씀(더 먹는 건 프로세스=대가). ② 진짜 이유 = **크래시 전파**: 스레드 방식은 탭 전부가 한 주소공간 공유 → 탭 A 크래시가 메모리 오염 → 전체 같이 죽음. 프로세스 방식은 격리 → A만 죽고 탭 B·나머지는 **멀쩡**("이 탭이 응답하지 않습니다"만 뜨는 그것).
  - **대가 ✅** — 프로세스는 무겁다·메모리↑. 보강: 격리돼서 탭 간 통신은 **IPC**라는 별도 통로 필요(공유 메모리보다 비쌈). 단어 주의: "처리속도 취약"이 아니라 **오버헤드(메모리·전환·통신)가 큼**(C 교훈 재확인, 연산이 느린 게 아님).
- **정답 정리 — 프로세스 선택 이유 2가지, 뿌리는 하나(격리)**:
  | | 장점 | 메커니즘 |
  |---|---|---|
  | 보안 | 악성 탭이 남의 탭 데이터 못 훔침 | 주소공간 격리 |
  | 안정성 | 한 탭 죽어도 나머진 산다 | 크래시 전파 차단 |
  | 대가 | 무겁다·메모리↑·IPC 필요 | 각자 주소공간 |

## 그날의 핵심
- **프로세스=격리(독립 주소공간), 스레드=공유(code·data·heap) + 각자 stack.** 이 한 줄이 모든 차이의 뿌리.
- **공유의 값과 대가**: 스레드는 데이터 나누기·전환이 싸지만 → race 위험 + 한 스레드 크래시가 전체를 죽임. 프로세스는 격리라 안전·안정하지만 → 메모리↑·통신 IPC 필요.
- **🔴 비용 ≠ 속도** (C): "스레드가 빠르다"는 부정확 → "생성·전환·통신 비용이 싸다"가 정확. T0 "커널모드=빠름"과 같은 함정 재발 — 내가 반복하는 실수로 등록.
- **크래시 격리**(크롬 탭): 안정성 차이는 메모리 용량이 아니라 **주소공간 공유 여부**(크래시가 전파되나/막히나)에서 온다.
