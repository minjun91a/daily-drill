# 🧠 OS 이론 (운영체제 원리) 트랙

[Linux 실무 트랙](../linux/README.md)·코드 드릴과 **별개**로, **운영체제 원리**를 기초부터 레벨 순서로. 명령 실습이 아니라 **개념**이라 드릴 형식이 다르다: 실행 대신 **정답 개념 대조**, 그리고 "그럴듯하지만 틀린 CS 설명 검증"(← daily-drill 핵심 테마와 동일). 트리거: **"오늘 OS"**.

## 형식 (개념 맞춤)
개념 씨앗(짧은 설명) → **예측/검증** 중 하나 —
- **오개념 검증**: "이 설명, 맞나 틀리나?" (그럴듯하지만 틀린 주장 찾기)
- **시나리오 예측**: "이 상황에서 OS는 어떻게 동작?"
- **트레이스**: 스케줄링·페이지 교체 등 손으로 계산

→ 정답 개념·추론으로 대조 → **직접**(개념을 내 말로 설명 = Feynman, 또는 계산 풀이). 가능하면 실제 코드(`fork`·스레드 경쟁 등)로 시연해 근거 제시.
